# MiniSNS GAS API 詳細設計書

## 1. プロジェクト概要
スプレッドシートをデータベースとして活用し、Ionic/Reactフロントエンドと連携する軽量SNSバックエンドシステム。

### ドライブ構成
- **MiniSNS/**
  - **MiniSNSDB** (スプレッドシート)
  - **BordAppAPIs** (GASプロジェクト)
  - **image/** (画像保存用フォルダ)

---

## 2. データベース（スプレッドシート）設計

### 2.1 UserInformation_Tbl (ユーザー情報)
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| UserID | string (PK) | ユーザーの一意識別子 |
| UserName | string(32) | 表示名 |
| Mail | string (PK) | メールアドレス |
| CreatedAt | datetime | アカウント作成日時 |
| Authority | string/number | 権限 (空 or 0~99) |
| UpdateAt | datetime | 最終更新日時 |
| DeleteFlag | number | 削除フラグ (0:有効, 1:削除済) |
| LoginDate | datetime | 最終ログイン/認証日時 |
| LoginToken | string (UK) | 疑似認証トークン (UUID) |
| password | string(32) | ハッシュ化済パスワード (SHA-256 + Salt) |
| ProfileTxt | string(500) | プロフィール文 |
| ProfileImageID | string | prf_N (Nはインクリメント) |

### 2.2 Bord_Tbl (投稿情報)
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| PostID | string (PK) | 投稿の一意識別子 |
| UserID | string (FK) | 投稿者のUserID |
| PostTxt | string(500) | 投稿内容 |
| PostDate | datetime | 投稿日時 |
| UpdateDate | datetime | 更新日時 |
| DeleteFlag | number | 削除フラグ |

### 2.3 Reply_Tbl (返信情報)
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| ReplyID | string (PK) | 返信の一意識別子 |
| PostID | string (FK) | 親投稿のPostID |
| UserID | string (FK) | 返信者のUserID |
| ReplyTxt | string(500) | 返信内容 |
| ReplyDate | datetime | 返信日時 |
| UpdateDate | datetime | 更新日時 |
| DeleteFlag | number | 削除フラグ |

## 3. GAS ファイル構成と役割

| ファイル名 | 役割 |
| :--- | :--- |
| **Main.gs** | APIエントリポイント。`doGet` / `doPost` によるルーティング。 |
| **Auth.gs** | ログイン、自動ログイン(トークン照合)、ユーザー登録処理。 |
| **BordOperation.gs** | 掲示板への投稿作成、取得、削除処理。 |
| **ReplyOperation.gs** | 投稿に対する返信の作成、取得、削除処理。 |
| **Common.gs** | ハッシュ化、レスポンス生成、トークン検証等の共通関数。 |
| **Config.gs** | シートID、フォルダID、バリデーション定数の定義。 |

## 4. 処理シーケンス

### 4.1 認証プロセス (Auth.gs)
トークン認証時には「5日ルール」を適用し、期限切れの場合は再ログインを促します。
```mermaid
sequenceDiagram
    participant FE as Frontend (Ionic)
    participant Main as Main.gs (doPost)
    participant Auth as Auth.gs
    participant DB as UserInformation_Tbl

    FE->>Main: POST { mode: "login/auto_login", ... }
    Main->>Auth: handleAuthentication()
    
    alt mode == "auto_login"
        Auth->>DB: トークンで検索
        alt 最終ログインから5日以上
            Auth->>DB: トークン破棄
            Auth-->>FE: error: 99 (Session Expired)
        end
    else mode == "login"
        Auth->>DB: ID/Mailで検索
        Auth->>Auth: UserIDをSaltにしてPassを照合
    end

    Auth->>Auth: 新UUIDトークン生成
    Auth->>DB: LoginDate, Token, UpdateAtを更新
    Auth-->>FE: status: ok, NewToken, UserInfo
